About Dataset

This dataset has been created by merging the census 2011 of Indian Cities with Population more than 1 Lac and City wise number of Graduates from the Census 2011. The Census is one of the biggest workout conducted by the Government of India to understand the demographics of the country. It is conducted every 10 years. The Britishers initiated it during the colonial times. It helps in a big way to come up with social and economic programs in the country. It also helps to identify the problems in the country. We can use this dataset to create a visualization to see where the future cities of India stand today. More information can be found

http://www.census2011.co.in

About the Visualization

This visualization project is following the Martini Glass narrative visualization structure. Here first page provides the details about the dataset and this page talks about the visualization. The next few pages provide the overview of Adult and Child population of various states in the country.

The first eight scenes provides the author driven content. Population Analysis page which is the last page contains reader driven content and it allows users to interact with data and draw their own conclusions.

All the chart are built using d3.js and HTML5

User Interface Events

User can navigate the pages by scrolling the mouse in up and down direction. User can click on the charts to get more details. Also, the last page provides the some user interaction when user clicks on the Geo map of India.

Annotations

The annotation is represented by the legends and tooltip, which provides the more information about the data.

Triggers

The mouse movement and mouse click events act as a trigger to faciliate the trigger on the charts. By moving the mouse over the charts tooltip appears and disappears. Also, on clicking on the GeoMap of India other charts are updated with the detailed information, which helps in drilling down the information on the visualization presentation.

Parameters

In the Analysis page, when user clicks on the GeoMap, the parameters has been passed from one chart to other chart to get the relevant details.
